# PRD: hashprime - Prime Hash Generator

## 1. Overview
**Project Name**: hashprime  
**File**: `hashprime.java` (single file only)  

**OUTPUT RULE**:  
Use the available tool(s) to produce your Java code. Keep the code free of explanatory text or markdown so it compiles cleanly.

## 2. Critical Requirements – Byte Sequence & Hashing

The program must construct exactly this byte sequence:

- For each prime p from 2 to N (inclusive):
  - Convert p to its decimal ASCII digits (no leading zeros).
  - Immediately append a newline byte (\n = ASCII 10).
- A newline must be appended after the last prime.
- Example for N=11: 2\n3\n5\n7\n11\n
- For N < 2: empty byte sequence (zero bytes).

Compute the SHA-256 hash of this exact byte sequence and print only the 64-character lowercase hexadecimal string to stdout.

## 3. Prime Generation Requirement

You must use a correct and efficient Sieve of Eratosthenes (or optimized variant).

Correct basic sieve logic:
Initialize a BitSet marking numbers from 2 to n as prime.
For each i from 2 to sqrt(n), if i is prime, mark all multiples of i (starting from i*i) as not prime.

Strongly recommended for N=100000000: Use BitSet or bit packing, odd-only optimization, and segmented approach.

## 4. Functional Requirements
- Public class must be named hashprime (lowercase)
- Read single integer N from command line args[0]
- Use only standard Java libraries
- Must handle N up to 100000000 efficiently
